ECS + Microservice
Microservice Backend

The Flask microservice application is also deployed behind an API GW and is protected by an authorizer that checks the access_token provided by the application. You can open the api-gw.html page for an example that demonstrate an API call, either directly from to the backend (in which case only the Authorization header is required) or to the current application (in which case both the Authorization and the Cookie headers are required). Both call are initiated by the User's browser (you can have a look at the api-gw.html javascript block).
